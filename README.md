# Semantic Search API with S-BERT and RAG/LLM

AI search engine built in Python, implementing two 
complementary approaches to intelligent text retrieval: semantic search 
using S-BERT and contextual question-answering using Retrieval-Augmented 
Generation (RAG) with Large Language Models.

This project applies real-world data from the 
California Government purchasing system.

## Project Overview

Searching through large volumes of contract and purchasing data is a 
challenge that goes beyond simple keyword matching. Language is imprecise 
by nature  the same concept appears under different terms depending on 
who wrote the record, when it was written, and what convention they followed.

This project addresses that gap. Rather than matching characters, it matches 
meaning. A query for "fuel" understands its relationship to "gasoline" and 
"diesel". A query for "seating" recognizes "chairs" and "benches" as 
contextually relevant. The search operates at the level of semantics.

This project is divided into two modules:

**Module 1 — Semantic Search with S-BERT**
Returns a ranked list of purchase records that are semantically similar 
to the search query, with a similarity score expressed as a percentage.

**Module 2 — Contextual Search with RAG and LLM**
Accepts a natural language question and returns a natural language answer, 
generated by a large language model using retrieved records as context.

## Data Pipeline

Before running the search API, the raw data goes through a multi-stage 
preprocessing pipeline:

1. Load 344,504 records from the California Open Data Portal CSV file
2. Remove irrelevant columns, drop null values and clean anomalies
3. Standardize column names using Python naming conventions
4. Filter out common stopwords such as "the", "and", "for", "in"
5. Reduce words to their root form, for example "ordering" becomes "order" and "diapers" becomes "diaper"
6. Convert all text to lowercase for consistency
7. Store the cleaned data in Apache Parquet format for fast retrieval

## How It Works

### Module 1: S-BERT Semantic Search

The user query goes through NLP preprocessing, then the S-BERT model 
converts it to a 768-dimensional vector. That vector is compared against 
all database vectors using cosine similarity, and results are returned 
ranked by similarity score.

The model used is `all-mpnet-base-v2`, trained on over one billion sentence 
pairs. It captures semantic meaning rather than matching surface-level keywords.

### Module 2: RAG Contextual Search

The user question is passed to FAISS, which retrieves the most relevant 
records from the corpus. LangChain then builds an augmented prompt combining 
the question with the retrieved context, and the LLM generates a concise 
natural language answer.

RAG solves the hallucination problem by grounding the LLM response in 
real records from the database rather than relying on the model's internal 
training data alone.

## Getting Started

### Prerequisites

Python 3.10 and a HuggingFace API key (free at huggingface.co)

### Installation

```bash
git clone https://github.com/YOUR_USERNAME/semantic-search-sbert-rag-llm.git
cd semantic-search-sbert-rag-llm

python3.10 -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt

python -m spacy download en_core_web_lg

python -c "import nltk; nltk.download('stopwords')"
```

### Environment Setup

Create a `.env` file in the root folder and add your token:
HUGGINGFACEHUB_API_TOKEN=your_token_here

### Run Data Preprocessing

```bash
python domain/purchases/PurchaseOrderPreprocessingDomain.py
```

### Generate Word Embeddings

```bash
python domain/purchases/PurchaseOrderWordEmbeddingsDomain.py
```

### Run Module 1

```bash
python application/SemanticSearchApi.py
```

Open Swagger UI at http://127.0.0.1:5000/apidocs and test with:

```json
{"sentence_query": "transport"}
```

### Run Module 2

```bash
python application/SearchRagApi.py
```

Open Swagger UI at http://127.0.0.1:5000/apidocs and test with:

```json
{"sentence_query": "Are there any computer services available?"}
```

## Key Capabilities

The application handles a wide range of real-world search scenarios:

Synonym detection means that searching "car" returns results for "automobile" 
and "vehicle". Plural and singular forms are treated as equivalent, so 
"chairs" matches "chair" at 100% score. The search is case insensitive, 
meaning "TRANSPORT" and "transport" return identical results. Stopwords 
are filtered automatically, so "Services and Maintenance" is treated the 
same as "Services Maintenance". Lemmatization reduces phrases like 
"ordering Diapers for the babies" to "order diaper baby" before searching. 
The engine also handles typos reasonably well, "vehice" successfully 
finds "vehicle". All results include a percentage similarity score, and 
the RAG module returns complete human-readable answers rather than just 
a list of records.
